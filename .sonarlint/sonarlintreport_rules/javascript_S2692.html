"indexOf" checks should not be for positive numbers (javascript:S2692)

Most checks against an indexOf call against a string or array compare it with -1 because 0 is a valid index. Any checks which look for values >0 ignore the first element, which is likely a bug.

Noncompliant Code Example

var color = "blue";
var name = "ishmael";
var number = 123;

var arr = [color, name];

if (arr.indexOf("blue") > 0) { // Noncompliant
  // ...
}
if (arr[0].indexOf("ish") > 0 { // Noncompliant
  // ...
}

Compliant Solution

var color = "blue";
var name = "ishmael";
var number = 123;

var arr = [color, name];

if (arr.indexOf("blue") >= 0) {
  // ...
}
if (arr[0].indexOf("ish") > -1 {
  // ...
}
